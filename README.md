# WoS\_OncoArticleClassifier

A collection of scripts and resources for processing metadata of Web of Science (WoS) articles on the application of artificial intelligence (AI) in oncology. The pipeline goes from raw Excel exports through deduplication, NLP-based filtering, binary classification, metric parsing and scoring, to summary statistics.

## 1. Data Preparation

### `combined_dataset.xlsx`

* **Contents**: \~45147 raw WoS records (1000 per Excel chunk) matching “cancer + artificial intelligence.”
* **Generated by**: a merge script on the `dev_draft` branch.
* **Fields**: DOI, Title, Abstract, Author Keywords, Publication Year, Journal, WoS Categories, etc.

### `processed_dataset.xlsx`

* **Contents**: Deduplicated articles (≈45147 → \~44964).
* **Generated by**: `dedupe_doi.py` on `dev_draft`.
* **Logic**: keep only the first occurrence of each DOI.

### `filtered_dataset.xlsx`

* **Contents**: \~19422 AI-oncology articles after relevance filtering.
* **Generated by**: **`data set filtration.py`**

  1. Loads `processed_dataset.xlsx`.
  2. Uses SpaCy + `task_keywords.csv` to remove non-relevant papers.
  3. Retains only articles whose title or abstract mention both AI and cancer.

## 2. Keyword Dictionaries (CSV)

All three CSV files drive SpaCy’s Matcher to assign binary flags.

| File                     | Purpose                                                                                                                                  |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **ai\_keywords.csv**     | Identify which AI/model type each paper uses (columns = model families: CNN, Transformer, Radiomics, etc.).                              |
| **cancer\_keywords.csv** | Identify cancer types studied (33 columns: breast, lung, prostate, colorectal, …).                                                       |
| **task\_keywords.csv**   | Identify research task (8 columns: Classification/Detection, Segmentation, Prognosis, Synthesis, Integration, NLP, Genomics, Auxiliary). |

---

## 3. Main Classification Script: `main_binary.py`

**Purpose**
Process `filtered_dataset.xlsx` from start to finish:

1. **Load** the filtered file.
2. **Binary classification** on three fronts:

   * **Cancer type** (via `cancer_keywords.csv`)
   * **AI model type** (via `ai_keywords.csv`)
   * **Task type** 8 categories (prognosis, segmentation, classification, etc.) (via `task_keywords.csv`)
3. **Parse performance metrics** in the Abstract:

   * Special handling for “test-group AUC” phrases.
   * General regex matching over 18 different performance metrics.
   * Fallback “proxy\_metric” for the first percentage found.
4. **Categorize** each metric into five levels (Very High, High, Medium, Low, Very Low) via `assign_category()` through task-specific thresholds: different performance expectations per task type
5. **Compute**:

   * `composite_metric`: the highest-priority metric present for that paper’s task.
   * `weighted_score` & `weighted_category`: weighted average of all available metrics, weights derived from the task’s priority list.
6. **Save** output as `filtered_dataset_binary_classification.xlsx`.

**Output columns**
All original fields plus:

* Binary flags: each cancer type, each AI model, each task.
* One column per metric (e.g. `precision`, `roc-auc`, …) holding the category label.
* Three new columns: `composite_metric`, `weighted_score`, `weighted_category`.

## 4. Post-Processing & Analytics

### `counter.py`

* **Purpose**: Read `filtered_dataset_binary_classification.xlsx`, compute summary tables:

  * Number of articles per cancer type.
  * Number of articles per AI model type.
  * Distribution of `composite_metric` and `weighted_category`.
* **Output**: `filtered_dataset_binary_classification_analysis.xlsx`.

### `countries_count.py`

* **Purpose**: Generate a country-level map of author affiliations from the final classification file.
* **Note**: This side analysis could live on `dev_draft` branch.

## 5. Branch `dev_draft`

Contains scripts for initial dataset assembly:

* **`merge_datasets.py`** – merge raw WoS exports into `combined_dataset.xlsx`.
* **`dedupe_doi.py`** – remove duplicate DOIs → `processed_dataset.xlsx`.
* Additional helpers.

## 6. Installation & Usage

1. **Prerequisites**:

   ```bash
   pip install pandas spacy tqdm openpyxl
   python -m spacy download en_core_web_sm
   ```
2. **Run main pipeline**:

   ```bash
   python main_binary.py
   ```
3. **Results**:

   * `filtered_dataset_binary_classification.xlsx`
   * Run `python counter.py` → `filtered_dataset_binary_classification_analysis.xlsx`
